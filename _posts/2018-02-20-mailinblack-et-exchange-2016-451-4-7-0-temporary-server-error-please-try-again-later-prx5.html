I. Présentation

Dans les versions Exchange 2016 avec MailinBlack, plateforme antispam,  vous pouvez occasionnellement rencontrer l’erreur suivante :

451 4.7.0 Temporary server error. Please try again later. PRX5 And in the connectivity logs you may see NS server returned ErrorRetry reported by 0.0.0.0. [Domain:Result] = server.domain.com:ErrorRetry (The DNS query for 'Undefined':'internalproxy':'00000000-0000-0000-0000-000000000000' failed with error : ErrorRetry)

ce qui à pour effet de bloquer les mails entrant dans la file d'attente sans jamais les délivrer aux boites utilisateurs.

Ceci est un problème qui peut être résolu avec la méthode suivante.

II. Procédure de dépannage

Par défaut, le connecteur de réception « Default Frontend » est lié sur toutes les adresses de la plage du serveur ce qui peut causer des erreurs avec le DNS Lookup.

Première étape : Connectez vous à votre centre d’administration Exchange.

Une fois connecté, dans le menu de gauche cliquez sur « Flux de messagerie »

Puis sélectionnez dans le menu « Connecteurs de réception »

Nous avons besoin de changer ici l’adresse IP de notre serveur Exchange 2016. Nous ajouterons plus loin aussi une ligne dans notre fichier « hosts ».

• Sélectionnez la ligne « (Toutes les adresses IPv4 disponibles) »
• Pour la modifier cliquez sur le symbole du crayon au-dessus.

• Dans la fenêtre qui s’ouvre, Sélectionnez le bouton radio « Spécifier une adresse IPv4 »
• Saisissez l’adresse IP de votre serveur Exchange dans la zone de texte en dessous.

Enregistrez les changements.

Vous revenez sur la page précédente et vous devez avoir le résultat suivant. Cliquer sur enregistrer pour conserver vos modifications.

Comme annoncé plus haut, la prochaine étape consiste à modifier le fichier hosts.

Pour cela vous devez vous connecter directement à votre serveur Exchange avec un compte possédant les droits d’administration c’est important.

Ouvrez le Panneau de configuration.

Allez dans Systèmes et sécurité -> Outils d’administration et lancez la console Services.

Arrêtez un à un les services Exchange qui sont en cours d’exécution.

Ouvrez l’explorateur Windows et naviguez jusqu’au fichier « hosts » qui se situe dans :

C:\Windows\System32\Drivers\etc

 

Faites un clique droit pour modifier le fichier (à l’aide du bloc note) et ajoutez les lignes correspondantes à l’adresse IP de votre serveur Exchange 2016 et à son nom d’hôte.

Pour notre exemple :

192.168.0.43 mail
192.168.0.43 mail.domain.local

Enregistrez vos modifications puis, pour terminer vous devez redémarrer votre serveur. Vous ne devrez plus rencontrer l’erreur : Exchange 2016 451 4.7.0 Temporary server error. Please try again later. PRX5.
